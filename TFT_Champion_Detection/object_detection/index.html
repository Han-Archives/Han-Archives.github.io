Step2. 영상을 프레임으로 나누기

기억해야 할 점

  1. 코덱이 있어야 동영상을 저장할 수 있다. 윈도우 os 상에서는 DIVX 코덱을 사용한다.
  2. VideoWriter에서 프레임 수를 지정할 수 있다.
  3. cap.get(1)으로 프레임수를 확인 및 mod를 사용하여 프레임 수를 제한할 수 있다.
  4. 윈도우 64bit에서 waitKey를 사용할 때 0xFF를 같이 사용한다.
import cv2
import os

os.chdir("C:\\Users\\USER\\Project\\2022\\Computer_vision")
## 동영상 1초당 2프레임으로 저장
cap = cv2.VideoCapture("video1.mp4")

# 코덱 정의
fourcc = cv2.VideoWriter_fourcc(*'DIVX') # window의 경우 DIVX

# 프레임 크기, FPS
width = round(cap.get(cv2.CAP_PROP_FRAME_WIDTH))
height = round(cap.get(cv2.CAP_PROP_FRAME_HEIGHT))

count = 0 # 프레임 번호 할당

while cap.isOpened():
    ret, frame = cap.read()
    
    if not ret:
        break
    
    # 1초당 1프레임을 png로 저장
    if(int(cap.get(1)) % 30==0):
        cv2.imwrite('C:\\Users\\USER\\Project\\2022\\Computer_vision\\Frames\\frame%d.jpg' % count, frame) # 0.jpg, 1.jpg
        count += 1
    
    frame = cv2.resize(frame, dsize=(0, 0), fx=0.5, fy=0.5, interpolation=cv2.INTER_AREA)
    cv2.imshow('video', frame)
    
    if cv2.waitKey(1)& 0xFF == ord('q'):
        break
        
cap.release()
cv2.destroyAllWindows()

Step4. Yolov5를 이용해 객체 인지 학습

아래의 과정은 Colab 환경에서 진행

Yolov5

  • Yolo란?

실시간으로 물체 인식이 가능한 모델로 객체 인식에 분야에서 제일 잘 알려진 모델

  • 특징

1.이미지 전체를 한번만 본다. Yolo가 나오기 전의 R-CNN 방법은 이미지를 여러개로 분할한 뒤 CNN 모델을 이용해 이미지를 분석했다. Yolo의 경우 이미지를 한번만 보는 특징을 지닌다. (프레임 속도 빠름)

2.통합된 모델을 사용한다. 기존의 방식의 모델은 다양한 전처리 모델과 인공 신경망을 결합한 뒤 사용. Yolo의 경우 통합된 모델을 사용한다. ( 더 간단하다.)

3.실시간으로 객체를 탐지할 수 있다. Yolo가 높은 성능을 가지는 것은 아니지만 준수한 성능으로 실시간 객체탐지가 가능하다.

  • 장단점

장점: 처리속도 빠르고 이미지를 한번 보는 방식으로 객체에 대한 맥락적인 이해도가 높고 테스트를 해도 다른 모델에 비해 높은 성능을 보인다.

단점: 다른 모델에 비해 낮은 정확도를 가진다. (특히 작은 객체)
